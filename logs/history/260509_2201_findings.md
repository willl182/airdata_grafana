# Technical Finding / Milestone

**Context**: Grafana downloader V2 TypeScript migration and A3 outputs.

## The Discovery

The V2 job pipeline now works as a typed TypeScript core with stable CommonJS CLI wrappers. Jobs can be resumed from existing raw JSON chunks and then finalized into user-facing artifacts without opening Chromium again when all chunks already exist.

A3 outputs are generated by `buildJobOutputs(job)` from `src/grafana/csv.ts`, which is called at the end of `runJob()` in `src/grafana/jobs.ts`. The primary output is a canonical long CSV. The optional wide CSV is controlled by `outputWide` in the job file.

Grafana field metadata matters for sensor names. The captured frames often use generic field names like `Value`, while the real station/sensor label is in `field.config.displayNameFromDS` and the frame schema name. The long CSV therefore records `frame_name`, `field_name`, and `field_key` so downstream API/webapp code can display station/sensor context correctly.
