# Technical Finding / Milestone

**Context**: Grafana downloader V2 Phase A2, job engine and resumable chunks.

## The Discovery

The downloader can now run from a job definition instead of only a global config. `src/grafana/jobs.js` loads a job JSON, normalizes config with existing defaults/env handling, parses `chunkSize`, generates deterministic chunks, writes job metadata, and executes only pending chunks.

The resumability model is file-based for this phase: if a chunk's expected raw JSON file already exists, that chunk is considered completed and is skipped. The skip is recorded in `manifest.jsonl` so repeated job runs leave an operational trace.

## Key Files Affected

- `src/grafana/jobs.js`: job loading, chunk generation, metadata writing, resumable `runJob`.
- `src/grafana/downloader.js`: added `runDownloadChunks(config, chunks)` to reuse one browser context for job chunks.
- `scripts/descargar-job.js`: CLI wrapper for job execution.
- `examples/job.7d.example.json`: 7-day example with daily chunks.
- `package.json`: added `download:job`.
- `README.md`: documented A2 usage and outputs.

## Validation

- `node -c` passed for changed and existing JS entry points.
- `examples/job.example.json` generates 1 chunk.
- `examples/job.7d.example.json` generates 7 chunks.
- `pnpm run download:job -- examples/job.example.json` completed successfully and wrote one raw JSON file.
- A second run of the 1-hour job reported `0 pendientes`.
- `pnpm run download:job -- examples/job.7d.example.json` completed successfully and wrote seven raw JSON files.
- A second run of the 7-day job reported `0 pendientes`.

## Operational Note

Playwright failed inside the assistant sandbox with:

```txt
sandbox_host_linux.cc:41 shutdown: Operation not permitted
```

Running `pnpm run download:job` outside the sandbox fixed it. Future validation of real Grafana downloads should use the approved `pnpm run download:job` prefix outside the sandbox.
