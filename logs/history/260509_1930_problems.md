# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Durante la implementacion aparecieron varios problemas tecnicos:

1. `npm install` funciono, pero el usuario pidio usar `pnpm`.
2. `pnpm install` fallo inicialmente porque intento escribir en el store global bajo `~/.local/share/pnpm`, fuera del area escribible del sandbox.
3. Playwright fallo al instalar navegadores en `~/.cache/ms-playwright`.
4. Chromium fallo dentro del sandbox con:

```txt
FATAL:content/browser/sandbox_host_linux.cc:41
shutdown: Operation not permitted
```

5. La primera exploracion con `waitUntil: "networkidle"` hizo timeout porque Grafana mantiene actividad de red y `networkidle` no fue un criterio estable.
6. La configuracion de prueba anterior dejo un JSON de 1 hora dentro del rango posterior de 7 dias, lo que puede duplicar datos al regenerar CSVs.

## User-Requested Corrections

- Usar `pnpm` en lugar de `npm`.
- Ejecutar la descarga para los ultimos 7 dias.
- Guardar la sesion, logs, aprendizajes y correcciones aplicadas usando el skill `saver`.

## The Solution / Workaround

1. Se eliminaron los artefactos de `npm` creados durante la primera instalacion:

```bash
rm -rf package-lock.json node_modules
```

2. Se instalo con `pnpm` usando un store local:

```bash
pnpm install --store-dir .pnpm-store
```

3. Se configuro Playwright para descargar Chromium dentro del proyecto:

```bash
PLAYWRIGHT_BROWSERS_PATH=/home/w182/w421/grafana/.ms-playwright pnpm exec playwright install chromium
```

4. Se actualizaron los scripts de `package.json` para usar:

```bash
PLAYWRIGHT_BROWSERS_PATH=.ms-playwright
```

5. Se ejecuto Playwright fuera del sandbox cuando fue necesario.

6. Se reemplazo `waitUntil: "networkidle"` por:

```js
waitUntil: "domcontentloaded"
```

y se agrego una espera por periodo quieto de respuestas relevantes.

7. Se configuro `config.local.json` para los ultimos 7 dias respecto al entorno local:

```txt
2026-05-02T19:26:00-05:00
a
2026-05-09T19:26:00-05:00
```

8. Se usaron ventanas de 1 dia para reducir riesgo de `502/503`.

## Learnings to Preserve

- Para dashboards normales de Grafana `/d/...`, no asumir endpoints de `/api/public/dashboards/...`; primero hay que capturar la red.
- En este caso el endpoint real fue `/api/ds/query?ds_type=influxdb`.
- `Inspect > JSON > DataFrame JSON (from Query)` es una referencia fuerte para validar el JSON capturado.
- `networkidle` no es fiable para dashboards Grafana; usar `domcontentloaded` mas una espera por quietud de respuestas de datos.
- En este sandbox, `pnpm` debe usar `--store-dir .pnpm-store` para evitar escrituras fuera del workspace.
- Playwright debe usar `PLAYWRIGHT_BROWSERS_PATH=.ms-playwright` para mantener navegadores dentro del proyecto.
- Ejecutar Chromium puede requerir permisos fuera del sandbox.
- El conversor actual lee todos los JSON de `data/raw`; si hay ventanas solapadas, los CSV incluiran duplicados.
