# Technical Finding / Milestone

**Context**: V2 product decisions and user workflow.

## The Discovery

The practical user workflow should not expose many JSON files or require ZIP handling by default. The app should download Grafana data in daily chunks internally, keep JSON raw as a technical artifact, and present one final CSV long file as the primary output. A CSV wide file remains useful as an optional convenience output.

The CSV long format is preferred because it is easier to process in R: one row per timestamp/sensor/value, with sensor identity in a column. It also scales better when many stations are present.

## Decisions Captured

- Maximum initial job range: 10 days.
- Default chunk size: 1 day.
- Primary output: CSV long.
- Optional output: CSV wide.
- JSON raw: internal artifact.
- ZIP: optional technical/debug artifact.
- First deployment access: Tailscale on port `3001`, no public domain initially.
