# Technical Finding / Milestone

**Context**: Grafana Downloader V2 planning

## The Discovery

The V2 should be organized as a cloud-operable job system rather than a simple long-running script. The critical design shift is:

```txt
script config -> job definition -> chunks -> worker -> artifacts
```

This allows large date ranges to run in small windows, survive failures, resume incomplete work, and expose progress in a web UI.

The recommended infrastructure remains VPS + Docker because Playwright/Chromium and long-running downloads fit poorly in typical serverless environments.

## Key Planning Decisions

- Use Hostinger VPS with Docker Compose for the initial deployment.
- Keep Vercel, Supabase, and Convex optional for later phases.
- Save JSON raw as the audit source.
- Generate CSV long as canonical output.
- Generate CSV wide for Excel-friendly inspection.
- Split execution planning into master, agent, and user plans.
