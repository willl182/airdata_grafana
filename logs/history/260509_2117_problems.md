# Problem Record (Troubleshooting)

**Severity**: Low

## The Problem

The initial V2 plan treated JSON, CSV long, CSV wide, and ZIP as equally prominent outputs. That would make the tool less ergonomic for the user because the normal workflow would create too many files and expose JSON that the user does not need to handle directly.

## User-Requested Corrections

- Set CSV long as the primary output.
- Keep CSV wide only as optional.
- Avoid making ZIP the main user workflow.
- Keep the initial max job range to 10 days because larger windows are known to be fragile.
- Use 1-day chunks by default.

## The Solution / Workaround

Updated the planning documents to make CSV long the primary deliverable, CSV wide optional, JSON internal, and ZIP technical/debug-only. The plan now explicitly limits V2 initial jobs to 10 days with 1-day chunks.

## Learnings to Preserve

- Optimize the app for the user's actual analysis workflow in R, not for exposing every intermediate artifact.
- Keep V2 operationally conservative: small chunks, small max range, resumable jobs.
- Technical artifacts should remain available for audit/debug, but they should not dominate the UI.
