# Technical Finding / Milestone

**Context**: Grafana dashboard data extraction

## The Discovery

El dashboard `https://grafana.canair.io/d/UN_OsIo7k/tangara` es un dashboard normal de Grafana, no un dashboard compartido bajo `/public-dashboards/<token>`.

La ruta viable fue automatizar lo que Grafana ya expone en `Inspect`:

- `Inspect > Data > Download CSV`
- `Inspect > JSON > DataFrame JSON (from Query)`

La exploracion de red con Playwright confirmo que el endpoint real para los datos del panel es:

```txt
POST /api/ds/query?ds_type=influxdb
```

Las respuestas contienen estructuras de Grafana DataFrame bajo `results[refId].frames`, con `schema.fields` y `data.values`.

## Implementation Milestone

Se implemento un proyecto Node con `pnpm` y Playwright:

- `scripts/explorar-grafana.js`: descubre requests relevantes y guarda muestras.
- `scripts/descargar-grafana.js`: descarga JSON crudo por ventanas temporales, con manifest, reintentos y reanudacion.
- `scripts/grafana-json-a-csv.js`: convierte DataFrames de Grafana a CSV por `refId`.
- `scripts/grafana-common.js`: utilidades compartidas.
