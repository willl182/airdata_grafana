# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem
Error persistente de parseo CSS en Next.js (Turbopack): `@import rules must precede all rules aside from @charset and @layer statements`. El error hacía referencia a una línea en `globals.css` que contenía `Share+Tech+Mono`, a pesar de que dicha línea ya había sido eliminada del archivo fuente.

## User-Requested Corrections
1. Cambiar el fondo a negro mate (remover texturas, grillas y scanlines).
2. Traducir términos clave al español (Jobs -> Descargas, Chunks -> Lotes, Artifacts -> Archivos).
3. Eliminar gradientes excesivos en barras de progreso.
4. Mantener la paleta de colores de acento (cyan y magenta/purpura).

## The Solution / Workaround
1. **Borrar Caché Turbopack:** El error de CSS se quedó "atascado" en la caché `.next` debido a un estado corrupto previo. Se ejecutó `rm -rf webapp/.next` y se reinició el servidor de desarrollo (`npm run dev`) para forzar una recompilación limpia.
2. **Refinamiento UI:** Se eliminaron las reglas `body::before` y `body::after` de `globals.css` para quitar las líneas horizontales y texturas CRT. Se cambiaron las barras de progreso a colores sólidos y se alinearon correctamente los elementos del header en la vista de detalle.

## Learnings to Preserve
1. **Next.js Turbopack Caching:** Turbopack puede ser agresivo con la caché. Si un error de compilación (especialmente CSS) parece desincronizado con el código actual, la solución inmediata es borrar el directorio `.next`.
2. **Aesthetic Restraint:** Evitar "AI slop" en diseños frontend (como excesivos gradientes en barras de progreso o animaciones ruidosas que sobrecarguen la UI). El usuario prefiere fondos limpios (matte black) con acentos limpios y puntuales.
