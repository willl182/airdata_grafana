# Technical Finding / Milestone

**Context**: Grafana Downloader V2 Phase A1

## The Discovery

The existing CLI downloader could be refactored into reusable CommonJS modules without changing the public commands. The safest A1 shape is:

```txt
scripts/* wrappers -> src/grafana/* reusable modules
```

This preserves the current operational workflow while creating the base needed for the future job engine.

## Key Files Affected

- `src/grafana/common.js`: shared config, URL, filesystem, frame detection, and request helpers.
- `src/grafana/downloader.js`: reusable `runDownload(config)` plus chunk capture functions.
- `src/grafana/explorer.js`: reusable `runExplore(config)`.
- `src/grafana/csv.js`: reusable `runCsv(config)` and CSV flattening helpers.
- `src/grafana/index.js`: public module export.
- `scripts/descargar-grafana.js`: compatibility wrapper for `pnpm run download`.
- `scripts/explorar-grafana.js`: compatibility wrapper for `pnpm run explore`.
- `scripts/grafana-json-a-csv.js`: compatibility wrapper for `pnpm run csv`.
- `scripts/grafana-common.js`: compatibility re-export.
- `examples/job.example.json`: initial example job for A2.
- `README.md`: documents structure and programmatic invocation.

## Validation

- `node -c` passed for all scripts and `src/grafana/*.js`.
- `node -e "const api = require('./src/grafana')..."` confirmed exported functions and example job loading.
- `pnpm run csv` completed successfully against existing local raw data.

## Learnings to Preserve

- Preserve `pnpm` scripts as the supported CLI entrypoints because they inject `PLAYWRIGHT_BROWSERS_PATH=.ms-playwright`.
- Avoid testing downloader wrappers via plain `node -e require(...)` unless the Playwright browser path is explicitly set.
- Keep user edits to `subplan_usuario_v2.md` separate from agent implementation commits unless explicitly requested.
